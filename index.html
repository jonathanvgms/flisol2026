Ventajas y Desventajas

El valor aparece cuando querés explorar rápido. El costo aparece cuando el prototipo empieza a pedir garantías.

Bueno

Velocidad y acceso

Idea a prototipo en minutos. Baja la barrera de entrada y sirve para demos, descartables y probar hipótesis.

No tan bueno

Riesgo invisible

Decisiones sin registro, código que no entendés y ningún contrato claro contra el cual verificar lo generado.

Ejemplo · Cupón

El prompt de un martes a las 17hs

Estoy armando el checkout de un e-commerce. Necesito una función JS: applyCoupon(cartTotal, coupon) que devuelva el total con el descuento aplicado. El cupón puede ser por porcentaje o por monto fijo.
Hacelo simple, después lo refino.
Ejemplo · Cupón

Lo que el agente devuelve

function applyCoupon(cartTotal, coupon) { if (coupon.type === 'percentage') { return cartTotal - (cartTotal * coupon.value / 100); } return cartTotal - coupon.value; }
applyCoupon(100, { type:'percentage', value:20 }) // → 80 ✅
Ejemplo · Cupón

Tres maneras de romperlo

1

Total negativo

applyCoupon(30, { type:'fixed', value:50 }) // → -20
Nunca dije que el descuento no pueda superar el total.
2

Cupón inmortal

// La función no chequea fecha. // Un cupón de Black Friday // sigue funcionando en marzo.
Nunca dije que los cupones venzan.
3

Reaplicación infinita

let t = 100; for (let i = 0; i < 5; i++) t = applyCoupon(t, { type:'percentage', value:50 }); // → 3.125
Nunca dije si se acumulan.
Conclusión

El agente no falló: faltaba el contrato

Total mínimo, vencimiento y acumulación no son bugs. Son reglas que nunca existieron para verificar.

# Feature: Aplicar cupón al carrito ## Reglas - Tipos: PERCENTAGE, FIXED_AMOUNT - Validar fecha de vencimiento (UTC) - Validar monto mínimo del carrito - Un solo cupón activo por carrito - El descuento no puede dejar el total bajo 0 - Redondeo HALF_UP a 2 decimales - Validar límite de uso por usuario ## Criterios de aceptación - Carrito $100 + cupón 20% → total $80.00 - Carrito $30 + cupón fijo $50 → total $0.00 (no negativo) - Cupón vencido → error VOUCHER_EXPIRED - Usuario ya usó el cupón → error VOUCHER_ALREADY_USED

Spec Driven Development

Desarrollar contra una especificación

SDD es usar una spec como contrato verificable: define qué debe pasar, qué límites existen y cómo saber si el resultado está bien.

  • Antes: intención, comportamiento y alcance.
  • Durante: el agente implementa guiado por la spec.
  • Después: se verifica contra criterios explícitos.

Ejemplo · Código fuente

La IA implementa contra el contrato

El resultado ya no es el happy path: el código refleja reglas, errores y bordes definidos en la spec.

applyCoupon.js
const ERR = { expired: 'VOUCHER_EXPIRED', alreadyUsed: 'VOUCHER_ALREADY_USED', multiple: 'ONLY_ONE_COUPON_ALLOWED' }; function applyCoupon(cart, coupon, ctx) { if (cart.activeCouponId && cart.activeCouponId !== coupon.id) throw domainError(ERR.multiple); if (new Date(coupon.expiresAt) < ctx.now) throw domainError(ERR.expired); if (ctx.usesByUser.get(coupon.id) >= coupon.limitPerUser) throw domainError(ERR.alreadyUsed); const subtotal = money(cart.total); const discount = coupon.type === 'PERCENTAGE' ? subtotal * coupon.value / 100 : coupon.value; return round2(Math.max(0, subtotal - discount)); }
Ya no adivina Lee reglas explícitas: vencimiento, uso por usuario y cupón único.
Falla con intención Devuelve errores de dominio, no resultados silenciosamente incorrectos.
Protege los bordes El total nunca baja de cero y el redondeo queda centralizado.

OpenSpec

Acordá qué vas a construir antes de escribir una sola línea de código.

Deltas revisables

Implementar SDD con OpenSpec

Del requerimiento al historial auditable, paso a paso

  1. 1

    Inicializar

    Estructura openspec/ y deja las instrucciones del agente dentro del repo.

    openspec init
  2. 2

    Proponer el cambio

    El agente redacta proposal, specs (deltas), design y tasks.

    openspec proposal
  3. 3

    Validar y revisar

    Aprobás el qué contra la spec antes de escribir una línea de código.

    openspec validate
  4. 4

    Implementar por tareas

    El agente ejecuta tareas chicas y revisables, marcando el progreso.

    tasks.md
  5. 5

    Archivar

    El delta se integra a las specs principales y queda historial auditable.

    openspec archive
