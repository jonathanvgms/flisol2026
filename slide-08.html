Spec Driven Development

La spec como fuente de verdad

  • Especificación clara de lo que el sistema debe hacer
  • Describir intención, comportamiento y alcance, y sirve para alinear a todos antes de construir

Implementar y verificar contra la spec

  • Guía la implementación y también para verificar si lo construido responde a lo pedido
  • El equipo escribe especificaciones claras del comportamiento esperado y luego genera, implementa o verifica código contra esas especificaciones

La spec como documento vivo

  • Una buena spec no se escribe una vez y se momifica
  • Se puede modificar, extender o reducir su alcance. Si la spec es muy grande posiblemente deba ser reducida en varias specs
